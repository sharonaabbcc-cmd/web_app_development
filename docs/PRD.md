# PRD - 景點收藏系統

## 1. 專案概述
- **背景與動機**：希望能有一個專屬的系統，用來記錄未來想去或曾經去過的景點，並能加上自己的筆記，打造個人化的旅遊知識庫。
- **目標用戶**：我自己單人使用。
- **核心價值主張**：提供一個輕量、簡單直覺的操作介面，讓我能快速新增、尋找景點，並且留下專屬心得筆記。

## 2. 功能需求
系統的主要功能與對應的使用者故事如下：

1. **新增景點**
   - 作為 **我自己**，我希望 **能夠透過簡單的介面輸入景點名稱、位置或相關資訊來新增景點紀錄**，以便 **未來規劃行程時能方便查看與參考**。

2. **尋找景點**
   - 作為 **我自己**，我希望 **能夠透過輸入關鍵字（如名稱或地點）來搜尋之前收藏的景點**，以便 **快速找出感興趣或規劃中需要的特定景點資訊**。

3. **新增與管理筆記**
   - 作為 **我自己**，我希望 **能針對收藏的每個景點加上專屬的個人筆記與感想**，以便 **完整記錄旅行行前規劃的重點或實際造訪後的心得**。

4. **景點分類與標籤**
   - 作為 **我自己**，我希望 **能為新增的景點加上不同的標籤或分類（例如：咖啡廳、大自然、展覽等）**，以便 **在尋找景點時能依據類別快速篩選**。

5. **編輯與刪除景點**
   - 作為 **我自己**，我希望 **能夠隨時修改景點內容或刪除不再需要的景點資料**，以便 **保持景點收藏清單的最新狀態與整潔**。

## 3. 非功能需求
- **技術限制**：
  - 伺服器端：基於 Python 的 Flask 框架開發。
  - 網頁前端：採用 Jinja2 模板引擎進行頁面渲染，並搭配 HTML/CSS/JS。
  - 資料庫：使用輕量級的 SQLite 作為關聯式資料庫。
- **效能與安全考量**：
  - **效能**：此工具主要為自我使用，不預期大流量，但需確保搜尋與整理速度的輕便性，目標頁面載入及操作回饋小於 1 秒。
  - **安全性**：對所有使用者輸入（如景點名稱、心得筆記）進行基本驗證或轉義處理，防止 XSS 與 SQL Injection 攻擊（透過 ORM 提供保護）。

## 4. MVP 範圍 (Minimum Viable Product)
| 階段層級 | 功能 | 說明 |
| :--- | :--- | :--- |
| **Must Have** | 新增景點 | 可以將想去的景點建檔。 |
| **Must Have** | 尋找景點 | 提供景點列表，以及關鍵字文字搜尋。 |
| **Must Have** | 新增與管理筆記 | 景點詳細頁面下方可以留存文字版筆記。 |
| **Should Have** | 編輯與刪除景點 | 可維護並過濾已造訪或不想去的景點。<br> |
| **Nice to Have** | 景點分類與標籤 | 提供可靈活設定的標籤進階篩選功能。 |
